# 📊 HH Vacancy Parser & Analyzer

Инструмент для сбора, очистки и анализа вакансий с hh.ru с использованием ML-классификации и интерактивного дашборда.

## Пайплайн

```
vacancy.csv (входной файл со ссылками)
      ↓
1_parse_vacancies.py       → собирает основные поля (название, зарплата, опыт, формат)
      ↓
2_parse_descriptions.py    → добирает описание и навыки по каждой вакансии
      ↓
3_clean_data.py            → очищает и нормализует все поля
      ↓
4_create_embeddings.py     → создаёт векторные представления текстов (Sentence Transformers)
      ↓
5_classify_and_analyze.py  → zero-shot классификация + кластеризация (K-Means + UMAP)
      ↓
6_dashboard.py             → интерактивный Streamlit-дашборд с фильтрами и редактором
```

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Подготовка входного файла

Перед запуском парсера нужно вручную подготовить файл `vacancy.csv` со ссылками на вакансии с hh.ru.

Формат файла:
```csv
url
https://hh.ru/vacancy/12345678
https://hh.ru/vacancy/87654321
```

Ссылки можно собрать вручную с сайта hh.ru или экспортировать из браузера.

### 3. Запуск по шагам

```bash
# Шаг 1 — парсинг основных полей
python 1_parse_vacancies.py

# Шаг 2 — парсинг описаний (медленнее, есть паузы против блокировки)
python 2_parse_descriptions.py

# Шаг 3 — очистка данных
python 3_clean_data.py

# Шаг 4 — создание эмбеддингов (нужен интернет, скачает модель ~400 МБ)
python 4_create_embeddings.py

# Шаг 5 — классификация и анализ
python 5_classify_and_analyze.py

# Шаг 6 — запуск дашборда
streamlit run 6_dashboard.py
```

### Промежуточные файлы (создаются автоматически, не хранятся в репозитории)

| Файл | Создаётся на шаге | Описание |
|------|-------------------|----------|
| `vacancy.csv` | вручную (входной) | ссылки на вакансии |
| `vacancy_parsed_v3_final.csv` | шаг 1 | основные поля |
| `vacancy_with_descriptions.csv` | шаг 2 | + описания и навыки |
| `vacancy_clean_v2.csv` | шаг 3 | очищенные данные |
| `vacancy_embeddings.npy` | шаг 4 | векторы (384D) |
| `vacancy_with_clusters.csv` | шаг 5 | + кластеры и UMAP-координаты |
| `vacancy_classified.csv` | шаг 5 | финальный файл с классификацией |

## Использованные технологии

| Инструмент | Для чего |
|------------|----------|
| `requests` + `BeautifulSoup` | парсинг страниц hh.ru |
| `pandas` | обработка данных |
| `Sentence Transformers` | векторизация текстов (`paraphrase-multilingual-MiniLM-L12-v2`) |
| `UMAP` | снижение размерности 384D → 2D |
| `K-Means` | кластеризация вакансий |
| `mDeBERTa-v3-base-xnli` | zero-shot классификация без обучающей выборки |
| `Streamlit` + `Plotly` | интерактивный дашборд |

## Переменные окружения

Для удобной работы с Hugging Face моделями создайте токен.

Получить токен можно на [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens) (бесплатно).

## Важные замечания

- Парсер работает **медленно намеренно** — между запросами стоят случайные паузы 4–10 секунд, чтобы не получить бан от hh.ru
- Часть вакансий может быть недоступна (удалены, в архиве, 403) — это нормально, скрипт обрабатывает такие случаи
- Модель для эмбеддингов скачивается автоматически при первом запуске (~400 МБ)
- Классификация работает на CPU, но при наличии GPU ускоряется автоматически
